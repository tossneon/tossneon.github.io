# Circle Heroes

SD 히어로 수집형 자동전투 방치형 게임. 모바일 APK로 설치해서 싱글플레이로 즐기는 걸 목표로 한다.

라이브(웹 프로토타입): `https://tossneon.github.io/circle-heroes/`

## 확정된 방향 (기획)

- **장르**: 방치형 키우기 + 자동전투. 환생/프레스티지 리셋 없음.
- **진행 구조**: 무한 스테이지 파밍 + 아레나(AI 임의 덱, 추후 실유저 매칭 확장 여지) + 탑 등반 + 업적.
- **영웅 획득**: 가챠(확률 뽑기) + 미션/스테이지 해금 혼합.
- **육성**: 레벨업 + 장비 강화(기본), 동일 등급 다른 영웅을 재료로 넣는 각성 합성, 진영/속성 시너지. 스킬트리 없음.
- **히든 요소**: Unknown 등급 히든 영웅 + 전용 히든 장비.
- **아트 무드**: SD/치비 스타일 (참고: 오늘도환생). 편성·전투 화면은 작은 SD 아이콘. 영웅카드/뽑기용 고퀄 초상화는 초기에는 만들지 않고 추후 확장 항목.
- **범위**: 싱글플레이 기준. 유저 계정/서버 쪽 유저별 데이터 분리 없음. APK를 지인에게 공유해도 각자 기기에서 독립적으로 진행.

## 기술 스택

- **게임 엔진**: Phaser 3 + TypeScript, 빌드는 Vite
- **모바일 빌드**: Capacitor (웹 → 안드로이드 APK)
- **세이브**: 기기 로컬 저장이 본체 (오프라인 동작, 매 전투마다 네트워크 왕복 없음)
- **마스터데이터(밸런스·영웅 스탯)**: 구글 스프레드시트를 "웹에 게시(CSV)"로 열어 무키(無key)로 가져옴 → 빌드 시 JSON으로 변환. 코드 수정 없이 시트에서 값 변경 가능.
- **백업**: Firebase. 클라이언트 설정은 공개되어도 안전하게 설계되어 있어 별도 프록시 서버가 필요 없음. 폰 교체/재설치 시 이어하기용, 지인 공유본은 선택적으로만 사용.

이 조합을 선택한 이유는 프록시 서버 같은 별도 인프라 없이, 로컬 우선(오프라인 완전 동작) + 노코드에 가까운 밸런싱(스프레드시트) + 가벼운 클라우드 백업(Firebase)을 모두 만족시키기 위함.
