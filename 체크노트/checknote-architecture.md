# 체크노트 — PWA 설계 문서

## 1. 전체 구조

```
[아이폰/안드로이드 브라우저]
        │  (홈 화면에 추가 → 앱처럼 실행)
        ▼
   React 웹앱 (Vite로 빌드)
        │
        ▼
   Firebase Firestore (실시간 DB)
        │
        ▼
[상대방 브라우저] ← 초대 링크로 접속
```

- **프론트엔드**: 지금 만든 React 프로토타입 그대로 재사용
- **백엔드**: Firebase (Google) — 서버를 직접 안 만들어도 됨, 무료 티어로 충분
- **배포**: GitHub 저장소 → Vercel 연결 → push할 때마다 자동 배포
- **로그인 화면 없음**: Firebase "익명 인증"으로 기기별 자동 식별 (아이디/비번 불필요)

---

## 2. 데이터 구조 (Firestore)

```
lists (컬렉션)
 └─ {listId} (문서)
     ├─ ownerId: "익명유저ID"
     ├─ partnerId: "상대방 익명유저ID" | null
     ├─ createdAt: timestamp
     └─ items (하위 컬렉션)
         └─ {itemId}
             ├─ text: "머리뿅 작은거"
             ├─ completedAt: timestamp | null
             ├─ createdBy: "익명유저ID"
             └─ createdAt: timestamp

invites (컬렉션)
 └─ {inviteCode}  예: "AB3XQ9"
     ├─ listId: "연결할 리스트 ID"
     ├─ createdAt: timestamp
     └─ used: false
```

**동작 흐름**
1. 앱 처음 실행 → Firebase 익명 인증으로 기기 고유 ID 자동 발급 (사용자는 아무것도 안 함)
2. 내 `lists` 문서 하나 자동 생성
3. "초대 링크 만들기" → `invites`에 6자리 코드 문서 생성 → `checknote.app/invite/AB3XQ9` 링크 완성
4. 상대방이 링크 클릭 → 자동으로 익명 인증 → 그 코드의 `listId`를 자기 partnerId로 연결
5. 이후 두 사람 다 같은 `lists/{listId}/items`를 구독(`onSnapshot`) → 한쪽이 추가/완료하면 **즉시 양쪽 화면에 반영**

---

## 3. PWA(홈 화면 앱)로 만드는 부분

- `manifest.json`: 앱 이름, 아이콘, 실행 시 주소창 숨김(`standalone` 모드) 설정
- 서비스워커: 오프라인에서도 마지막으로 본 리스트는 뜨게 캐싱
- 아이폰: 사파리에서 링크 열고 "공유 → 홈 화면에 추가" 한 번만 하면 진짜 앱처럼 아이콘 생김
- 안드로이드: 크롬이 자동으로 "홈 화면에 추가" 배너를 띄워줌

---

## 4. 로드맵 (순서대로)

| 단계 | 내용 | 예상 소요 |
|---|---|---|
| 1 | Vite + React 프로젝트로 지금 코드 이식 | 30분 |
| 2 | Firebase 프로젝트 생성, Firestore + 익명인증 켜기 (콘솔 클릭 몇 번) | 20분 |
| 3 | 항목 추가/완료 로직을 로컬 state → Firestore 연동으로 교체 | 1~2시간 |
| 4 | 초대 링크 생성/수락 로직 구현 | 1시간 |
| 5 | manifest.json + 아이콘 추가로 PWA화 | 30분 |
| 6 | GitHub에 push → Vercel 연결(저장소 import만 하면 자동 배포) | 15분 |
| 7 | 실제 폰 두 대로 초대 테스트 | - |

git 계정 있으시니 4~7단계는 각 잡고 진행하면 반나절 안에 실제 작동하는 링크가 나와요.

---

## 5. 비용

- **Firebase**: 무료(Spark) 티어로 충분. 읽기/쓰기 하루 5만 회, 저장공간 1GB까지 무료 — 개인+배우자 사용 규모면 절대 안 넘음
- **Vercel**: 개인 프로젝트 무료 티어로 충분
- **도메인**: 없어도 `xxx.vercel.app` 기본 주소로 계속 써도 됨. 나중에 원하면 `.com` 하나 사서 연결(연 1~2만원)

**총비용: 사실상 0원**으로 시작 가능해요.
