# Windows 스크립트 규칙

이 PC 는 Windows 10 + **Windows PowerShell 5.1**(pwsh 아님)이고, 작업 경로에 `★경환폴더`(별표+한글)가 들어 있다. 아래는 전부 실제로 발생했던 사고에 대한 대응이다.

## 인코딩

- PowerShell 5.1 은 **BOM 없는 UTF-8 을 시스템 코드페이지(CP949)로 읽어 한글을 깨뜨린다.** 스크립트 파일은 반드시 **UTF-8 BOM** 으로 저장한다.
- **스크립트 안에 한글 리터럴을 넣지 않는다.** 경로·이름 같은 한글 값은 **인자로 전달**받는다.
  - 예: `backup-side-projects.ps1` 이 repo 경로를 하드코딩하지 않고 `-Repo <path>` 인자로 받는 이유가 이것이다. 단순화한다고 경로를 스크립트 안에 박으면 백업이 조용히 실패한다.
- 파일을 쓸 때 `Out-File`/`Set-Content` 의 기본 인코딩은 UTF-16 LE 다. 다른 도구가 읽을 파일이면 `-Encoding utf8` 을 명시한다.
- 콘솔 출력에서 한글이 깨져 보이는 것은 콘솔 표시 문제일 뿐, 변수에 담긴 값 자체는 정상 유니코드인 경우가 많다. 출력만 보고 로직 버그로 오판하지 말 것.

## 경로

- `★경환폴더` 같은 특수문자 경로에서 **외부 프로그램 실행이 실패할 수 있다** (`spawn UNKNOWN`). 크롬을 이 경로의 프로필로 띄우려다 실패한 전례가 있다.
- 경로에 공백·특수문자가 있으므로 항상 큰따옴표로 감싼다. 공백 포함 exe 는 호출 연산자로 실행한다: `& "C:\Program Files\App\app.exe" arg`

## 문법

- PowerShell 5.1 에는 `&&`, `||`, 삼항연산자(`?:`), null 병합(`??`)이 **없다.** `;` 와 `if ($?) { }` 를 쓴다.
- 네이티브 exe 에 `2>&1` 을 쓰지 않는다. 5.1 은 stderr 를 ErrorRecord 로 감싸서 exit code 0 인데도 실패로 판정한다.
- 대화형 명령(`Read-Host`, `Get-Credential`, `pause`)은 쓰지 않는다 — 자동 실행 시 멈춘다.
