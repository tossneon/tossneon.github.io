# 버크만 자동화 (birkman-automation)

버크만 진단 **구매대행 + AI 디브리핑**을 자동화하는 도구.
네이버 스마트스토어 주문 → 버크만코리아에서 진단지 구매·발송 → 결과 PDF 확보 → AI 디브리핑 생성까지.

## 단계 (Phase)

| Phase | 범위 | 상태 |
|---|---|---|
| **1. 진단지 구매대행** | 로그인 세션 재사용 → 진단지 구매 → 대상자(이름·이메일) 등록 → 재발송 대응 → 결과 PDF 다운로드 | 흐름 매핑 완료 |
| **2. AI 디브리핑** | 결과 PDF → 구조화 → Claude 해석 생성 → 자격자(사용자) 검수 → 전달 | **파이프라인 작동(v1)** |
| **3. AI 챗봇** | 대상자 결과 기반 Q&A 챗봇 링크 제공 | 아이디어 |

## 주문 처리 런북 (주문 1건 처리 순서)

1. **로그인** — 앱 내장 브라우저에서 사용자가 버크만 로그인(세션 유지). 비번은 사용자만 입력.
2. **구매** — `/assessment/view?code=...` 에서 진단지 결제 (⚠ 실제 결제, 확인 후 진행).
3. **대상자 발송** — `/mypage/assessment` → 해당 주문 **전송** → 대상자 이름·이메일 입력.
4. **완료 대기** — 대상자가 검사 완료하면 상태가 "진단완료"로.
5. **결과 다운로드** — 주문 클릭 → 대상자 상세 → 진단파일 **다운**.
   - 직접 주소: `/mypage/download/assessment/each?num={대상자ID}`
6. **디브리핑 생성** —
   ```bash
   node src/extract-pdf.mjs "결과.pdf" "out.txt"      # PDF→텍스트
   node src/parse-report.mjs "out.txt" "out.json"      # 텍스트→구조화 JSON
   # out.json + templates/debriefing-prompt.md → Claude로 디브리핑 생성(.md) → 자격자 검수
   node src/make-debriefing-pdf.mjs "디브리핑.md" "디브리핑.pdf"   # 마크다운→첨부용 PDF
   ```
7. **메일 발송** — 대상자에게 디브리핑 PDF 전달. (버크만 원본 리포트는 버크만이 대상자에게 자동 발송)
   - **미리보기(안전)**: `node src/send-debriefing.mjs --to <이메일> --name <이름> --pdf "디브리핑.pdf"`
   - **실제 발송**: 위 명령 끝에 `--send` 추가 (사전에 `.env` 설정 필요 — `.env.example` 참고)
   - 또는 Gmail 초안만 만들고 수동 검토·발송 (첨부는 손으로).

> ⚠ 실제 발송(`--send`)은 고객에게 나가는 행위이므로, 디브리핑 검수 후에만 실행하세요.

> ⚠ 마이페이지는 직접 URL 접근 시 비번 재확인 게이트가 뜸. 자동화에선 내부 메뉴 이동으로 최소화.

> 스토어 개설·상품등록(앞단)과 구매확정·AS·매출관리(뒷단)는 **사업관리 영역**으로 보고, 우선 서비스 동작(위 Phase 1~2)을 먼저 만든다.

## 🔐 보안 원칙 (중요)

- 이 저장소(`tossneon.github.io`)는 **공개**다. 민감 데이터는 절대 커밋 금지.
- **코드만** 커밋. 세션/쿠키·고객정보·결과 PDF 는 `data/`(gitignore)에만 저장.
- **비밀번호는 코드/AI가 다루지 않는다.** 로그인은 사용자가 브라우저에서 직접(최초 1회), 세션은 영속 프로필에 유지.
- **진단지 구매 = 실제 결제.** 구매 단계에는 확인 게이트 / dry-run 을 둔다.

## 사용법

```bash
npm install            # 최초 1회 (playwright 설치)
npx playwright install chromium

npm run login          # 최초 1회: 브라우저에서 직접 로그인 → 세션 저장
npm run map <URL>      # 정찰: 특정 페이지의 폼/버튼 구조 덤프 (셀렉터 확정용)
npm run run            # (구축 예정) 주문 처리 오케스트레이터
```

## 폴더

```
src/
  browser.js   영속 프로필로 브라우저 실행 (로그인 세션 재사용)
  login.js     최초 1회 수동 로그인 부트스트랩
  map.js       페이지 구조 정찰(셀렉터 확정용)
data/          [gitignore] 세션·고객정보·결과 PDF
```
